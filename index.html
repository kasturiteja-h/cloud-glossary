A beginner-friendly glossary of essential cloud terms

Object Storage

A service to store website files (HTML, CSS, JS, images). Example: AWS S3, Azure Blob, Google Cloud Storage.

Content Delivery Network (CDN)

A global caching system that makes websites load faster. Example: AWS CloudFront, Azure CDN, Google Cloud CDN.

DNS (Domain Name System)

Translates domain names (like mywebsite.com) into the cloud service where your website is hosted. Example: AWS Route 53, Azure DNS, Google Cloud DNS.

SSL/TLS

Provides HTTPS security for your website. Example: AWS Certificate Manager, Google Managed SSL.

CI/CD (Continuous Integration & Deployment)

Automates the process of deploying website changes from GitHub or other repos. Example: AWS CodePipeline, Azure DevOps, Google Cloud Build.

IAM (Identity and Access Management)

A cloud service that manages **who can access what**. It controls permissions for users, groups, and roles. Example: AWS IAM, Azure Active Directory, Google IAM.

Serverless Computing

A model where you run code without managing servers. You only pay for execution time. Example: AWS Lambda, Azure Functions, Google Cloud Functions.

Cloud Monitoring

Services that track performance, errors, and usage of cloud apps. Example: AWS CloudWatch, Azure Monitor, Google Cloud Monitoring.

Load Balancer

A service that distributes traffic across multiple servers to keep apps fast and reliable. Example: AWS Elastic Load Balancing, Azure Load Balancer, GCP Load Balancer.

Virtual Machine (VM)

A software-based computer running inside a cloud server. You can install OS and apps just like on a physical PC. Example: AWS EC2, Azure Virtual Machines, Google Compute Engine.

Containers

Lightweight environments that package apps and dependencies together. Easier to run, move, and scale. Example: Docker, Kubernetes.

Database as a Service (DBaaS)

Cloud-based databases that are fully managed for you. No need to install or manage servers. Example: AWS RDS, Azure SQL Database, Google Cloud SQL.
